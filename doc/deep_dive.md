## 1. 物理问题背景

### 1.1 场景：推力轴承

想象两块平行圆盘夹着一层极薄的润滑油。下面那块盘表面刻有**螺旋槽**（类似黑胶唱片的纹路，但是螺旋形状）。当盘高速旋转时（6000 rpm），油被螺旋槽"泵"向中心，在两盘之间形成高压油膜，将上盘"托"起来——这就是**流体动压润滑**。

```
          ┌─────────────────────┐   ← 上盘（光滑平面，静止）
  油膜    │   ░░░ 润滑油 ░░░░  │   ← 膜厚仅 3 μm（人头发的 1/20）
          └─────────────────────┘   ← 下盘（刻有螺旋槽，高速旋转）
               ↗ 螺旋槽放大
          ┌──┐     ┌──┐
  槽截面  │深│ ... │深│  ← 槽深 = 3×膜厚, 周期=6 个扇区/360°
          └──┘     └──┘
```

**核心矛盾**：膜太薄（3μm）→ 承载力巨大，但稍有偏差就会导致两盘接触磨损。所以必须精准预测油膜的压力分布。

### 1.2 Reynolds 方程：这个问题的"牛顿第二定律"

完整的 Navier-Stokes 方程（流体力学的基本方程）在这个场景下可以大幅简化，因为：

- 膜厚方向的尺度（μm） << 径向/周向尺度（mm），相差 1000 倍
- 惯性力相对于粘性力可忽略（层流）

简化后得到 **Reynolds 方程**——它是推力轴承润滑问题的"薛定谔方程"：

```
 1    ∂  ⎛      3  ∂P ⎞    1   ∂  ⎛  3  ∂P ⎞        ∂H          ∂(γ·H)
─── · ── ⎜R · H  · ───⎟ + ─── · ── ⎜H  · ───⎟ - Λ · ───  + Λ · ────── = 0
 R    ∂R ⎝         ∂R ⎠    R²   ∂θ ⎝      ∂θ ⎠        ∂θ           ∂θ
 \_______________________/   \______________________/   \_____/   \__________/
       泊肃叶流（扩散项）         泊肃叶流（扩散项）      楔效应     空化效应
```

用普通话解释四项物理含义：

| 项 | 物理含义 | 类比 |
|----|---------|------|
| **项1+2**（扩散项，`H³·∇P`） | 压力梯度驱动流体流动——高压区流向低压区 | 像热量从高温传到低温 |
| **项3**（楔效应，`Λ·∂H/∂θ`） | 槽几何形状变化引起"泵吸"——膜厚收敛区产生高压 | 像楔子把油"铲"进窄缝 |
| **项4**（空化项，`Λ·∂(γH)/∂θ`） | 空化导致膜厚被气泡"充填"，改变有效流动面积 | 像水管里有气泡，水流规律变了 |

**类比理解**：Reynolds 方程本质上是一个**守恒方程**（质量守恒）——流入和流出每个微小控制体的油量必须平衡。前三项是"常规流动"，第四项是"空气中产生的修正"。

### 1.3 空化（Cavitation）：为什么油会"冒出气泡"

#### 物理原因

当局部压力降到润滑油的**蒸气压力**（接近真空）以下时，液体无法维持，溶解的气体析出形成气泡。在推力轴承中，螺旋槽的发散区（膜厚突然增大）会产生负压趋势，触发空化。

#### JFO 空化模型

**JFO**（Jakobsson-Floberg-Olsson）是质量守恒型空化模型，是工程界最广泛接受的处理方式。它将解域分为两个区域：

```
全膜区 (Full Film):             空化区 (Cavitation):
  油完全填满间隙                    油中析出气泡
  P > 0,    γ = 0                 P = 0,    γ > 0
  ↗ 压力为正                     ↗ 压力为零（蒸气压力）
  ↗ 空化率为零                    ↗ 空化率 > 0（气泡占比）
```

**空化率 γ (gamma) 的物理含义**：
- γ = 0：该点完全被液体油填满
- γ = 1：该点完全是气泡（极端情况）
- γ ∈ (0,1)：油气混合物（工程中最常见）

**互补条件** `P · γ = 0`：任何一个点上，压力和空化率不能同时为正——要么有压力（全膜），要么有空化（气泡），不存在"又有压力又有气泡"的状态。

#### 为什么用 Fischer-Burmeister 函数？

JFO 的互补条件是**非光滑**的（分段函数），直接用 PINN 求导困难。**Fischer-Burmeister (FB) NCP 函数**提供了一种光滑松弛：

```
P + γ - √(P² + γ²) = 0
```

- 当 P > 0, γ → 0 时：`P + 0 - P = 0` ✓
- 当 P → 0, γ > 0 时：`0 + γ - γ = 0` ✓
- 当 P, γ 均非零时：函数值 > 0（就是不满足约束）

把它作为一个**额外损失项**来惩罚"物理上不可能"的状态。

### 1.4 所有参数逐个详解

> "这个数为什么是这个值？"——每个参数的物理由来。

#### 几何参数

| 参数 | 符号 | 代码值 | 单位 | 通俗解释 |
|------|------|--------|------|---------|
| 内径 | r_i | 47.0×10⁻³ | m | 轴承内圆半径，油从这儿供入 |
| 外径 | r_o | 52.0×10⁻³ | m | 轴承外圆半径，油从这儿流出 |
| 平衡膜厚 | h_i | 3.0×10⁻⁶ | m | 无槽区域的间隙高度（3μm = 头发的1/20） |
| 周期数 | K | 6 | — | 整个圆周上螺旋槽重复几次（360°/6=60° 每个扇形） |
| 槽起始位置比 | R_d_1_ratio | 1.043 | — | 槽的起始半径 / 内径，槽从 r≈49mm 处开始 |
| 槽结束位置比 | R_d_2_ratio | 2.212 | — | 槽的结束半径 / 内径，槽延伸到 r≈104mm（实际 = 外径外，即完全贯通） |
| 螺旋角 | α | 3.0° | deg | 槽与径向的夹角，决定螺旋线的形状（对数螺线） |
| 槽深比 | h_texture_ratio | 3.0 | — | 槽深 = 3×3μm = 9μm（槽底比台面深 3 倍膜厚） |
| 槽宽比 | groove_ratio | 0.5 | — | 槽宽占一个周期的 50%（一半是槽，一半是台面） |

#### 工况参数

| 参数 | 符号 | 代码值 | 单位 | 通俗解释 |
|------|------|--------|------|---------|
| 内径压力 | p_i | 0.1×10⁶ | Pa | 0.1 MPa，供油压力（约 1 个大气压） |
| 外径压力比 | p_o_ratio | 1.5 | — | p_o = 1.5×p_i = 0.15 MPa（外径稍高，防止倒流） |
| 空化压力 | p_c | 0.0 | Pa | 蒸气压力（假设为绝对真空，简化） |
| 动力粘度 | η | 8.00×10⁻⁴ | Pa·s | 润滑油的"粘稠度"（≈水的 800 倍） |
| 转速 | ω | 6000 | rpm | 下盘旋转速度（100 转/秒） |
| 参考压力 | p_base | 10×p_o | Pa | 无量纲化基准压力（=1.5 MPa） |

#### 无量纲轴承数 Λ 的含义

```python
Lambda = (6 · η · ω · r_o²) / (h_i² · p_base)
       = (6 × 8e-4 × 628.3 × 0.0027) / (9e-12 × 1.5e6)
       ≈ 600
```

**物理含义**：Λ 是"楔效应强度"与"泊肃叶扩散强度"的比值。
- Λ 大（高速/高粘度）→ 楔效应主导 → 压力集中
- Λ 小（低速/低粘度）→ 扩散效应主导 → 压力平缓

Λ ≈ 600 意味着楔效应很强，轴承承载能力大（但也更容易出现空化）。

### 1.5 膜厚函数 H(R, θ)：螺旋槽的数学表示 —— 为什么非用 sigmoid 不可？

#### 螺旋槽的几何定义

膜厚不是均匀的——螺旋槽让它随 (R, θ) 变化。基本逻辑：

- 如果点 (R, θ) 落在**槽区**（groove）→ 膜厚 = 1.0 + 3.0 = 4.0（深 4 倍）
- 如果点 (R, θ) 落在**台面区**（land）→ 膜厚 = 1.0（平衡膜厚）

但"落在槽区"怎么判断？需要三个条件同时成立：

```
① 径向位置在槽范围内:   R_d_1 < R < R_d_2
② 周向位置在槽宽度内:   θ_sym(R) < θ < θ_sym(R) + groove_width
③ 周期性:               上述条件对每个 60° 扇形都成立（通过加 offset 实现）
```

其中螺旋线的形状由**对数螺线**定义（代码 `reynold_pinn.py` 第 139-141 行）：

```python
def theta_sym(R):
    return tf.math.log(R / r_g) / tf.math.tan(alpha) + theta_offset
```

这是一个对数螺线——螺旋槽不是直的斜线，而是随半径增大而弯曲的曲线，这样能把油更高效地泵向中心。

#### Sigmoid 出现在哪里？——逐行对应代码

代码在 `reynold_pinn.py` 的 `create_H_func` 中（第 143-158 行）。关键部分是 `is_texture` 的计算：

```python
# 代码第 149-150 行：周向判断（点在槽宽内吗？）
term = (tf.math.sigmoid((theta - theta_sym(R) + offset) / xi_theta) *
        tf.math.sigmoid((theta_sym(R) - theta + 2*pi/K*groove_ratio - offset) / xi_theta))

# 代码第 153-155 行：径向判断 + 合并
is_texture = (tf.math.sigmoid((R - R_d_1) / xi_R) *
              tf.math.sigmoid((R_d_2 - R) / xi_R) *
              sum(periodic_terms))

# 代码第 157 行：最终膜厚
H = 1.0 * (1 - is_texture) + (1.0 + h_texture/h_i) * is_texture
```

每个判断都用了 `sigmoid(差值 / ξ)`，其中 ξ（xi）控制过渡宽度：

| 参数 | 值 | 含义 |
|------|-----|------|
| ξ_R | (R_max-R_min) / 100 ≈ 0.00096 | 径向过渡宽度 = 域宽的 1% |
| ξ_θ | (θ_max-θ_min) / 50 ≈ 0.021 | 周向过渡宽度 = 域宽的 2% |

#### 为什么必须平滑？—— 自动微分的硬需求

这不仅仅是为了"好看"。Reynolds PDE 残差中包含 H 的**导数**：

```python
# reynold_pinn.py 第 171-183 行，PDE 残差中的 H 相关项：
part_1  = tf.gradients(R * H**3 * p_R, R)[0] / R    # ← 需要 ∂H/∂R
part_2  = tf.gradients(H**3 * p_theta, theta)[0] / R**2  # ← 需要 ∂H/∂θ
part_3_1 = -Lambda * tf.gradients(H, theta)[0]       # ← 需要 ∂H/∂θ
part_3_2 = -Lambda * tf.gradients(-gamma * H, theta)[0]  # ← 需要 ∂H/∂θ
```

PINN 用 `tf.gradients`（自动微分）计算这些项。如果 H 是一个**阶跃函数**（硬阈值 if-else）：

```
         H
         ↑
    4.0  │         ┌────────┐
         │         │ 槽区   │
    1.0  │─────────┘        └─────────
         │   台面              台面
         └──────────────────────────→ θ
              ↑               ↑
           边界1           边界2
```

在槽的边界处，H 是**不连续的**。自动微分会产生：
- `∂H/∂θ` → **无穷大**（Dirac delta 函数），导致 PDE 残差爆炸
- `∂²H/∂θ²` → **NaN**（delta 函数的导数），梯度传播崩溃
- 损失函数在边界附近无法优化（"悬崖"地形）

用 sigmoid 替代后：

```
         H
         ↑
    4.0  │              ,───────╮
         │            ╱          ╲
    1.0  │──────────╱             ╲──────────
         │
         └──────────────────────────→ θ
              ← ξ_θ 过渡带 →
```

现在 H 是 **C∞（无限可微）** 的，`tf.gradients` 在任何点都能正常工作。

**类比**：这就好比你用 ReLU 网络训练时，虽然 ReLU 在 x=0 处不可微，但在实际使用中靠子梯度勉强可用。但 Reynolds 方程需要**二阶导数**（对 H³ 求两次导），连 ReLU 都撑不住——必须用 sigmoid 这样的光滑函数。

#### 过渡宽度 ξ 的权衡

ξ 太大 → 槽边界模糊，物理精度下降
ξ 太小 → 接近阶跃，梯度数值不稳定

当前设置（ξ_R ≈ 域宽 1%，ξ_θ ≈ 域宽 2%）是在物理精度和数值稳定性之间的经验平衡。这个值是手工调的，本身也是一个可优化的超参数。

### 1.6 计算域的"巧思"：为什么只算 1/6 个圆？

整个圆盘有 K=6 个完全相同的扇形周期。利用旋转周期性，只需计算一个 60° 扇形：

```
计算域:  R ∈ [r_i/r_o, 1] ≈ [0.904, 1.0]
         θ ∈ [0, 2π/K]      = [0, 60°]
```

这大大缩小了问题规模，是工程上常见的对称性简化。

## 2. 对比模型

### 2.1 FEM（有限元法）：传统的"金标准"

FEM 用 **FBNS (Fischer-Burmeister NCP Solver)** 求解器，核心思路：

```
① 将计算域离散为 201×201 网格（40,401 个节点）
② 在每个单元上用形函数插值
③ 建立弱形式 → 组装刚度矩阵
④ 用 Newton-Raphson 迭代求解非线性系统
⑤ JFO 空化条件嵌入为 NCP（非线性互补问题）
```

**优点**：

- 成熟可靠，是工程设计的标准工具
- 精度可控（加密网格即可提高精度）
- 201×201 分辨率下误差极小

**缺点**：

- 需要**人工划分网格**（螺旋槽的复杂几何使得网格划分困难）
- 对复杂几何（如非规则槽形）适应性差
- 求解非线性系统计算量大（每步 Newton 迭代都需要组装+求解大型稀疏矩阵）
- 空化自由边界需要特殊的数值处理（正则化、松弛等）

**FEM 数据来源**：`p_FBNS.txt` 和 `g_FBNS.txt` 就是这个 FEM 求解器的输出——每行 `[R, θ, value]`，共计 201×201=40401 行。该项目将其作为**ground truth**。

### 2.2 PINN（本项目）：一种替代范式

PINN 用神经网络直接参数化解：

```
传统 FEM:  网格 → 离散化 → 求解线性代数系统 → 得到节点值
PINN:      随机配点 → 自动微分计算 PDE 残差 → 最小化损失 → 得到连续函数
```

**核心差异**：

| 维度 | FEM | PINN |
|------|-----|------|
| 解的形式 | 网格节点上的离散值 | 连续可微函数 `NN(R,θ) → [P, γ]` |
| 空间离散 | 需要网格划分 | **无网格**（只需随机配点） |
| PDE 满足方式 | 弱形式（积分方程） | 强形式（微分方程残差） |
| 边界条件 | 显式施加 | 损失项惩罚 或 网络硬约束 |
| 非线性求解 | Newton 迭代 | Adam + 梯度下降 |
| 推理 | 插值 | 直接前向传播 |
| 泛化 | 仅限网格范围内 | 可在任意坐标点预测 |

**为什么 PINN 值得尝试？**

1. **无网格**：不需要针对螺旋槽几何生成复杂的结构化网格
2. **可微解**：自动提供压力和空化率的任意阶导数（FEM 需要后处理）
3. **逆问题友好**：可在同一框架下进行参数辨识（例如从实测反推膜厚）
4. **连续表示**：可在任意分辨率下评估（本项目的 401×401 可视化远超 FEM 的 201×201）

## 3. PINN 模型详解

### 3.0 模型输入与输出（先明确基本概念）

在看架构之前，必须搞清楚"这个网络到底接收什么、输出什么"。

#### 输入

网络接收一个 shape 为 `(N, 2)` 的张量：

```
输入 = [R, θ]

其中:
  R  ∈ [0.904, 1.0]    ← 无量纲径向坐标（物理半径除以 r_o = 52mm）
  θ  ∈ [0, 1.047]      ← 无量纲角度坐标（= 60° = 2π/6，一个扇形周期）
```

**物理含义**：输入就是轴承环形计算域内的一个点的位置坐标。注意这不是全圆盘，而是只取了 K=6 个周期中的 **1 个 60° 扇形**（利用旋转对称性），R 也只取了环形区域的内外径之间。

> 在 Coslayer_normalization 层内部，R 和 θ 会被进一步归一化到 [-1, 1]。

#### 输出

网络输出是一个 **list of 2 个 tensor**，每个 shape 为 `(N,)`：

```
输出 = [P, γ]
  ↑    ↑
  │    └── 空化率 (cavitation fraction)，默认≥0，物理范围 [0, 1]
  └─────── 无量纲压力，默认≥0，本算例范围约 [0, 0.1]
```

| 输出 | 物理含义 | 数值范围 | 如何保证 |
|------|---------|---------|---------|
| **P** | 无量纲压力 `p/p_base`。P>0 意味着该点油膜有承载能力 | 本算例约 [0, 0.1]（即物理压力 0~0.15 MPa），边界固定为 P_i=0.0667, P_o=0.1 | `tanh²` 硬约束非负 + 硬约束 BC 精确指定边界值 |
| **γ** | 空化率。γ=0 全油膜，γ=1 全是气泡，中间值油气混合 | [0, 1] | `tanh²` 硬约束非负 + sigmoid 可替换 |

**注意**：P 和 γ 不是独立输出的——它们在网络中间共享大部分隐藏层，只在最后的线性层分出两个头。这意味着网络学习的是 P 和 γ 的**共享物理表示**（它们通过 JFO 互补条件强耦合）。

#### 完整的"输入→输出"示意图

```
[R, θ]  两个标量，代表计算域内一个点的极坐标
  │
  ▼
┌──────────────────────────────────────────┐
│           PINN 神经网络                   │
│  Coslayer → 4×128 隐藏层 → 双头输出       │
│  + 硬约束BC嵌入 + tanh²输出激活            │
└──────────────────────────────────────────┘
  │
  ▼
[P, γ]  两个标量，分别代表该点的压力和空化率
  │
  ▼
代入 PDE 残差: f_PDE(R,θ) = Reynolds方程残差
代入 FB 条件: f_FB(R,θ) = P+γ-√(P²+γ²)

训练目标: f_PDE→0 且 f_FB→0（在所有配点上）
```

---

### 3.1 网络架构改进全景图

这个 PINN 不是"裸"MLP——它在网络结构中嵌入了**大量物理先验**。下面逐一剖析每项改进的设计动机：

#### 改进 1：Coslayer_normalization（余弦归一化层）

**动机**：角度方向有天然的 **2π 周期性**。普通的 MLP 输入 `(R, θ)` 对 θ 没有周期性感知——`θ=0.01` 和 `θ=2π-0.01` 在物理上几乎相同，但 MLP 会把它们看成完全不同的输入。

**方案**：在输入层用 `cos(θ·π + φ)` 对角度坐标做傅里叶特征映射：

```
输入 [R_raw, θ_raw]
  ↓
归一化: R = 2(R_raw-R_min)/(R_max-R_min)-1 → [-1, 1]
        θ = 2(θ_raw-θ_min)/(θ_max-θ_min)-1 → [-1, 1]
  ↓
傅里叶映射: cos(θ·π + φ_trainable)  ← 可训练的相位参数
  ↓
输出 = w_R·R + w_θ·cos(θ·π + φ) + bias
```

**效果**：
- `cos(θ+2π)` = `cos(θ)` → 周期性自动满足，无需额外损失项
- 可训练参数 φ（相位）允许网络自适应地调整周期性特征的"对齐"

---

#### 改进 2：硬约束边界条件（Hard BC Enforcement）—— 逐行代码拆解

这是整个网络架构中**最精巧但也最难理解**的设计。我把它拆成 5 个问题逐步解释。

---

##### Q1: 为什么要"硬约束"？软约束有什么问题？

传统 PINN 把边界条件作为**损失项**来惩罚（软约束）：

```
L_total = L_PDE + λ · L_BC

其中 L_BC = MSE(P_pred(边界点), P_true(边界值))
```

问题在于权重 λ 极其难调：
- λ 太小 → 边界条件不满足 → 解完全偏离
- λ 太大 → 网络只关心边界 → PDE 残差被忽略 → 解无物理意义

**硬约束**的思路：与其"惩罚"网络让它别犯错，不如直接在**网络结构**里把边界条件焊死——无论权重怎么更新，边界处的输出永远不会错。

---

##### Q2: 硬约束的基本公式是什么？

核心公式只有一行：

```
P_final(R,θ) = g_func(R) + σ_func(R) · NN_pred(R,θ)
                ↑               ↑            ↑
           边界插值函数     距离函数      网络原始输出
           (决定边界值)   (边界处=0)   (只有内部起作用)
```

**直觉**：
- 在边界上（R = -1 或 1）：`σ_func = 0` → `P_final = g_func`（只由插值函数决定，网络"失声"）
- 在内部（-1 < R < 1）：`σ_func > 0` → `P_final = g_func + 网络修正`（网络可以自由调整）

这样边界条件被**数学上精确满足**，而非"尽力接近"。

---

##### Q3: `g_func` 到底怎么构造的？为什么有 `atanh` 和 `sqrt`？

这是最关键的细节。让我从后往前推导。

**最终输出是什么？** 代码第 480 行：

```python
predictions = (tf.nn.tanh(predictions))**2    # ← P_final = tanh(raw)²
```

这意味着无论 `raw` 是多少，`P_final` 永远 ≥ 0（`tanh² ∈ [0, 1)`）。这是为了物理合理性——负压力没有物理意义。

**问题来了**：如果最终输出 = `tanh(某值)²`，那么在边界上我们希望：
```
P_final(R=-1) = P_i = 0.0667    ← 内边界压力
P_final(R=+1) = P_o = 0.1000    ← 外边界压力
```

这要求：
```
tanh(raw)² = 0.0667   →   tanh(raw) = √0.0667   →   raw = atanh(√0.0667)
tanh(raw)² = 0.1000   →   tanh(raw) = √0.1000   →   raw = atanh(√0.1000)
```

**这就是 `atanh` 和 `sqrt` 的来源！** `atanh` 是 `tanh` 的反函数——因为输出要经过 `tanh²`，所以必须提前把边界值"反推"回去：

```
物理空间:   P_i = 0.0667  ──√──→  0.2582  ──atanh──→  0.264  ← 这是网络内部需要的值
物理空间:   P_o = 0.1000  ──√──→  0.3162  ──atanh──→  0.327  ← 这是网络内部需要的值
```

验证：`tanh(atanh(√P_i))² = (√P_i)² = P_i` ✓

**代码对应**（第 453 行）：

```python
g_func_1 = tf.math.atanh(
    (√P_o - √P_i) / 2.0 * (R + 1.0) + √P_i
)
#  ↑
#  在 R=-1 时: (√P_o-√P_i)/2 × 0 + √P_i = √P_i  → atanh(√P_i)
#  在 R=+1 时: (√P_o-√P_i)/2 × 2 + √P_i = √P_o  → atanh(√P_o)
#  这是标准的线性插值: value(R) = (value_1-value_0)/2×(R+1) + value_0
```

---

##### Q4: `g_func_2` 又是干什么的？

`g_func_1` 只是**线性插值**——它假设边界值之间的内部走势是直线。但真实物理解的内部走势可能更复杂。`g_func_2` 是一个**可学习的 Hermite 插值修正项**（第 462 行）：

```python
g_func_2 = (x_1 * (R + 1) * ((R - 1)/(-2))² 
         +  x_2 * (R - 1) * ((R + 1)/( 2))²)
#          ↑                          ↑
#     NN 输出的可训练系数          NN 输出的可训练系数
```

检查边界：
- R = -1：`(R+1)=0` → 第一项 = 0；`(R+1)/(2)=0` → 第二项 = 0 → **g_func_2 = 0** ✓
- R = +1：`(R-1)=0` → 第二项 = 0；`(R-1)/(-2)=0` → 第一项 = 0 → **g_func_2 = 0** ✓

所以 `g_func_2` 在边界上**恒为零**——它永远不会破坏边界条件，但允许网络在**内部自由塑造边界附近的形状**。这是二点三次 Hermite 插值多项式，x_1 和 x_2 对应边界处的**斜率**，由网络学习得到。

最终：`g_func = g_func_1 + g_func_2`（第 464 行）

---

##### Q5: `σ_func` 的距离函数有什么特别？

`σ_func` 使用 `Out_Imp_BC_layer`（第 445 行），它不是固定公式，而是**可训练的指数型距离函数**：

```python
σ(R) = p₃ · (1 - exp(p₁·(-1-R))) · (1 - exp(p₂·(R-1)))
#       ↑          ↑                         ↑
#    可训练      项1: R=-1时=0              项2: R=+1时=0
```

验证边界：
- R = -1：`p₁×(-1-(-1)) = 0` → `e⁰=1` → `1-1=0` → **σ=0** ✓
- R = +1：`p₂×(1-1) = 0` → `e⁰=1` → `1-1=0` → **σ=0** ✓
- 内部：两个指数项都 < 1 → σ > 0（正值可训练调整）

**为什么用可训练的指数函数，而不用固定的 `(1-R²)/√(...)`？**

因为指数衰减速率由 p₁, p₂ **可学习**——网络可以自动决定"多快从边界过渡到内部"，这是一个让边界层厚度自适应于物理问题的巧妙设计。例如空化边界附近的梯度很陡，可能需要更厚的过渡层。

---

##### 完整的前向传播追踪

把所有拼在一起，跟踪一个 `(R=-1, θ=任意)` 的边界点走完网络：

```
输入: [R=-1, θ=0.5]
  ↓ Coslayer_normalization → 特征向量 x, inputs_R=-1
  ↓ 4层隐藏层 (U/V混合) → 隐藏特征
  ↓ Dense(1) → predictions (NN原始输出)
  ↓ Dense(1) → prediction_g (γ的原始输出)

--- P 的构造 ---
g_func_1 = atanh((√P_o-√P_i)/2 × (-1+1) + √P_i) = atanh(√P_i) ≈ 0.264
g_func_2 = x_1 × 0 × (...)² + x_2 × (...) × 0² = 0
g_func   = 0.264 + 0 = 0.264
σ_func   = p₃ × (1 - e^{p₁×0}) × (...) = p₃ × 0 × (...) = 0
P_raw    = 0.264 + 0 × NN_pred = 0.264
P_final  = tanh(0.264)² = 0.2582² = 0.0667 = P_i  ← 精确！

--- γ 的构造 ---
γ = tanh(σ_func₂ × prediction_g)² = tanh(0 × ...)² = tanh(0)² = 0
  ← 边界上 γ=0（边界条件：边界处无空化）
```

现在跟踪一个内部点 `(R=0, θ=0.5)`：

```
g_func_1 = atanh((...)/2 × 1 + √P_i) = atanh(中间值)   ← 线性插值的中间点
g_func_2 = x_1 × 1 × (1/4)² + x_2 × (-1) × (1/4)²     ← 网络学习的修正
σ_func   = p₃ × (1-e^{-p₁}) × (1-e^{-p₂}) > 0           ← 边界层外, σ>0
P_raw    = g_func_1 + g_func_2 + σ_func × NN_pred       ← 网络可以自由调整
P_final  = tanh(P_raw)²                                  ← 非负约束
```

**一句话总结**：硬约束 BC 通过"反函数预补偿 + 零边界乘法"技巧，让边界条件在网络结构内精确成立，把"满足边界条件"从优化问题变成了**架构不变式**。

---

#### 改进 3：双分支可学习混合（U/V Mixture of Branches）

**动机**：压力和空化率是两个物理性质迥异的场（压力连续光滑、空化率在空化边界处剧烈变化），共享同一个特征空间可能导致"互相迁就"。

**方案**：每层维护两个独立分支 U 和 V，通过可学习的门控权重进行混合：

```
x_new = tanh(gate(x) ⊙ U(x) + (1-gate(x)) ⊙ V(x))
         ↑            ↑                    ↑
      混合权重     分支U输出             分支V输出
```

这相当于每层有一个"路由器"决定每个神经元走 U 还是 V 路径，给予网络更多自由度来同时表达 P 和 γ 的不同特征。

---

#### 改进 4：τ 稳定化项（Cavitation Boundary Stabilization）

**动机**：空化边界是 PDE 解的**内边界**（自由边界，解自身决定位置），在这里 P 和 γ 都趋于 0，Reynolds 方程的椭圆性退化，数值不稳定性极高。

**方案**：在 PDE 残差中添加自适应人工扩散项：

```
f_PDE = Reynolds_standard + ∂²γ/∂θ² · τ · τ₂

τ   = stop_gradient( (|∂γ/∂θ| - ∂γ/∂θ) · ε )    ← γ 增长时 > 0
τ₂  = stop_gradient( (∂P/∂θ  - |∂P/∂θ|) · ε )    ← P 下降时 > 0
ε   = 0.1
```

**关键设计**：`stop_gradient()` 使 τ 和 τ₂ 在反向传播时不被视为函数，只作为**系数**乘在二阶导数上。这确保了稳定化项不改变梯度流向，仅起到"自适应各向异性扩散"的作用。

**物理直觉**：当压力在下降（`∂P/∂θ < 0`）且空化率在增长（`∂γ/∂θ > 0`）时——这正是越过空化边界的标志——τ·τ₂ > 0，此时添加一个正比于 `∂²γ/∂θ²` 的扩散来平滑数值震荡。

---

#### 改进 5：PCGrad 多任务梯度冲突消解

**动机**：这是一个多任务学习问题——PDE 残差、FB 互补条件、边界条件构成 4 项损失函数。如果梯度方向相互冲突（内积 < 0），优化器会"左右为难"，收敛缓慢甚至振荡。

**PCGrad 核心思想**（Projecting Conflicting Gradients）：

```
对任意两个任务的梯度 g_i, g_j:
  if g_i · g_j < 0:               ← 梯度方向冲突
      g_i = g_i - (g_i·g_j)/(g_j·g_j) · g_j
            ↑                        ↑
        原始梯度              在 g_j 上的投影分量（去掉它）
```

几何解释：如果两个梯度夹角 > 90°，把 g_i 投影到 g_j 的**正交补**上——只保留 g_i 中与 g_j "不冲突"的分量。这保证了每次参数更新时，没有一个任务在"倒退"。

**为什么随机打乱梯度顺序？** PCGrad 的结果依赖于处理梯度的**顺序**。随机打乱后再投影，确保公平性（没有哪个任务被系统性偏袒）。

---

#### 改进 6：RAD 残差自适应配点细化

**动机**：随机均匀分布的配点在 PDE 残差大的区域（空化边界附近）可能不够密集。类比 FEM 的自适应网格细化（AMR）。

**RAD 流程**：

```
步骤 1: 生成密集测试网格（10×N_f 个点）
步骤 2: 在测试网格上计算 PDE 残差 → r²
步骤 3: 将 r² 归一化为概率分布
步骤 4: 按概率采样 N_add = ratio × N_f 个新配点
步骤 5: 合并到配点集 → 下一轮训练
步骤 6: 重置配点集到基础规模 → 重复（避免过度膨胀）
```

**参数设置**：`ratio_RAD_list = [0.03, 0.01]` 分别对应 PDE 残差和 FB 互补条件的加点比例，总共约为 3% + 1% = 4% 的新增配点。

└─────────────────────────────────────────────────────┘

**设计意图**：

1. **分段衰减 LR**：每阶段内 LR 从大到小分 3 档（PiecewiseConstantDecay），前期大步快速收敛，后期小步精细调优
2. **每阶段降低初始 LR**：阶段 1 从 1e-3 开始（粗调），阶段 4 从 1e-5 开始（精调），逐步收窄搜索范围

4. **Adam only（不用 L-BFGS）**：`newton_iter=0`，完全依赖一阶优化器。L-BFGS 在某些实验中可能更不稳定（尤其对空化边界的锯齿型残差）
5. **Adam β₁=0.99**：略高于默认值 0.9，让一阶动量更"记旧"，有助于在 RAD 改变配点分布后保持训练连续性

## 4. 测评方法

### 4.1 为什么需要测评？

PINN 的训练**没有 ground truth**——它只有 PDE 物理约束（残差最小化），但 PDE 残差小 ≠ 解正确（病态问题可能有多个解满足 PDE）。因此需要和已知可靠的**参考解**（FEM）做对比。

### 4.2 测评数据来源

| 数据 | 来源 | 格式 |
|------|------|------|
| FEM 压力 P | `p_FBNS.txt` | 201×201 网格，FBNS 求解器 |
| FEM 空化率 g | `g_FBNS.txt` | 同上 |
| PINN 预测 | `output/checkpoints/epochs_best_model` 加载后推理 | 在相同的 201×201 网格点上评估 |

### 4.3 误差指标体系

#### 全局场误差

| 指标 | 公式 | 本实验值 | 说明 |
|------|------|---------|------|
| **P_rel_L2** | ‖P_FEM - P_PINN‖₂ / ‖P_FEM‖₂ | **1.39×10⁻²** | 压力的全局相对误差 1.4% |
| **P_rel_Linf** | max\|ΔP\| / max\|P_FEM\| | 7.22×10⁻² | 压力最大偏差约 7.2% |
| **P_MAE** | mean\|ΔP\| | 3.84×10⁻⁴ | 压力平均绝对偏差 |
| **P_RMSE** | √mean(ΔP²) | 6.86×10⁻⁴ | 压力均方根偏差 |
| **G_rel_L2** | ‖g_FEM - g_PINN‖₂ / ‖g_FEM‖₂ | **1.18×10⁻¹** | 空化率的全局相对误差 11.8% |
| **G_MAE** | mean\|Δg\| | 1.14×10⁻² | 空化率平均绝对偏差 |

**解读**：

- 压力场精度**较高**（~1.4% L2），这在工程上是可接受的
- 空化率精度**中等**（~11.8%），空化边界的准确定位对 PINN 仍是挑战

#### 分区域误差（空化区 vs 全膜区）

| 指标 | 本实验值 | 解读 |
|------|---------|------|
| P_rel_L2_fullRegion | **1.33×10⁻²** | 全膜区压力误差很小 |
| P_rel_L2_cavRegion | 8.57×10¹⁴ | ⚠️ 空化区压力相对误差极大 |
| G_rel_L2_fullRegion | 1.20×10¹⁷ | ⚠️ 全膜区空化率相对误差极大 |
| G_rel_L2_cavRegion | **7.35×10⁻²** | 空化区空化率误差可控 |

**解读（重要！）**：

- 空化区 P ≈ 0（参考值），分母是 ‖P_FEM_cav‖ ≈ 0 → 导致相对误差爆炸（**并非模型崩溃**，而是除法数值问题）
- 全膜区 g ≈ 0（参考值），同理分母极小 → G_rel_L2_fullRegion 的 10¹⁷ 也是除法问题，不是实际误差大
- **真正有意义的指标**：全膜区 P 误差 1.3%（✓）和空化区 g 误差 7.4%（△）

#### 空化区域形状匹配

| 指标 | 公式 | 本实验值 | 说明 |
|------|------|---------|------|
| **Cavitation IoU** | \|交集\| / \|并集\| | **0.271** | PINN 空化区与 FEM 空化区的重叠度 |
| **Cavitation Dice** | 2\|交集\| / (\|A\|+\|B\|) | **0.426** | 同上，更宽松的度量 |

**解读**：IoU=0.27 偏低，说明 PINN 预测的空化边界与 FEM 有显著偏差。这是本模型的主要弱点——精确的空化边界定位仍然困难。FEM 有 201×201 的分辨率优势，而 PINN 仅用 ~5000 个配点。

#### 物理一致性检查

| 指标 | 公式 | 本实验值 | 说明 |
|------|------|---------|------|
| **互补条件违反量** | max(P · g) | **3.20×10⁻³** | JFO 条件的"违反度"（理想值=0） |

**解读**：最大 P·g ≈ 3.2×10⁻³ 意味着存在极少量点同时具有非零 P 和 g，但量级很小，Fischer-Burmeister 互补条件基本被满足。

#### 周期性一致性

| 指标 | FEM | PINN |
|------|-----|------|
| P 周期偏差 | 极小 | 需检查 |
| g 周期偏差 | 极小 | 需检查 |

Coslayer_normalization 理论上应保证周期性，对比脚本的 `plot_periodic_check` 用于验证这一设计的实际效果。

## 5. 总结：这个项目的贡献与局限

### 贡献

1. **首次将 PINN 应用于 JFO 空化 Reynolds 方程**（据代码推判）
2. **多项网络架构创新**：硬约束 BC、余弦周期层、τ 稳定化、PCGrad
3. **完整的 FEM 对比验证**：9 组可视化 + 6 类指标

### 局限（基于测评结果推断）

1. **空化边界精度不足**（IoU=0.27）：在 4900 个配点下难以精确复现 FEM 201×201 网格的空化边界
2. **空化区压力不稳定**：相对误差爆炸（尽管可能是除法问题），需要更稳健的评估
3. **仅对比单一工况**：只有一组几何参数 + 一组工况（6000 rpm），缺乏参数敏感性分析
4. **PINN 的训练成本**：80000 迭代 × Adam，与 FEM 的求解时间对比没有定量报告
